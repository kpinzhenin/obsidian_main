Для использования дебаггера необходимо компилировать gcc с флагом -g и без оптимизации
```bash
gcc -g -o someSource someSource.c
```
отладчик - это отдельная программа. Поэтому запускается 
```bash
gdb someSorce
gdb someSorce 1234 # specify process ID
gdb -p 1234        # for debug runned process
```
Для начала отладки используется команда [[run]], в уже запущенном gdb
```bash
(gdb) run
```
Shell команды остаются доступными внутри gdb.
Опции команд задаются через `-`, и если они имеют параметр до он устанавливается после `--`

Прерывание исполнения производится:
	breakpoint - безусловная остановка в опеределнных местах
	watchpoint - условная остановка, если "меняется переменная из выражения". пишут, что работает медленнее чем breakpoint, но используется, в момент, когда не понятно в каком месте возникает ошибка. 
	catchpoint - остановка происходит при возникновении определенных событий (сигналы и прочее)

Прерывание управляются несколькими командами:
	[[Break or b]] - создание
	[[Clear]] - удаление
	[[enable]] - разрешение использования
	[[disable]] - запрещение использования

Для запуску программы на исполнение используется команда run или r. Выполнение происходит до точки останова.дальнейшее исполнение производится командой n
